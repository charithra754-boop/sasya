# Acceptance Criteria: SasyaAI

This document defines the technical and functional Acceptance Criteria (AC) that must be met before any feature or microservice in the **SasyaAI** ecosystem is marked ready for production deployment.

---

## 1. Dynamic Digital Twin & Ingestion Pipelines

### AC-1.1: User Profile and Identity (FR-1.1)
* **AC-1.1.1:** Given a new farmer registration, the system must validate the AgriStack ID format and retrieve the corresponding land coordinates and owner name from the AgriStack Sandbox API within 1.5 seconds.
* **AC-1.1.2:** If the AgriStack ID is invalid, the API must return a `400 Bad Request` status code with a localized message: *"Invalid AgriStack ID, please verify."*

### AC-1.2: Data Fusion and Refresh Latency (FR-1.2, FR-1.4)
* **AC-1.2.1:** The Apache Kafka ingestion pipeline must process IMD weather data and eNAM mandi prices, synchronizing the records to the PostgreSQL Digital Twin database within 15 minutes of the feed becoming available at the source.
* **AC-1.2.2:** Duplicate event inputs from Kafka streams must be discarded using idempotent database write transactions based on unique event identifiers.
* **AC-1.2.3:** Soil Health Card data updates must trigger audit log insertion in MongoDB, capturing: `timestamp`, `updated_by`, `previous_npk_values`, and `new_npk_values`.

---

## 2. Multi-Agent Orchestration & AI Services

### AC-2.1: LLM Router & Query Processing (FR-2.1)
* **AC-2.1.1:** The LLM Orchestrator must extract query intents (e.g., `CROP_PLANNING`, `DISEASE_DIAGNOSIS`, `SCHEME_ENQUIRY`) with a classification accuracy of >95% on a validation set of 1,000 randomized user queries.
* **AC-2.1.2:** Response generation latency for the Orchestrator, excluding voice synthesis, must not exceed 2.5 seconds under a concurrent load of 500 active sessions.

### AC-2.2: Vision Agent - On-Device Pest Diagnosis (US-1.2, FR-2.3)
* **AC-2.2.1:** The quantised YOLOv8 pest diagnosis model must be packaged into a binary size of < 15MB to allow installation and caching on devices with limited storage.
* **AC-2.2.2:** The on-device inference execution time on an Android mobile CPU (e.g., MediaTek Helio G35) must be under 800 milliseconds.
* **AC-2.2.3:** The model must achieve a Mean Average Precision (mAP50) of >= 85% for the 20 major target pests on the NPSS dataset.
* **AC-2.2.4:** If offline, image diagnostics must cache locally in SQLite. The background sync manager must upload cached records to the central Postgres database within 10 seconds of detecting network availability (`status: online`).

---

## 3. Decision Engine & Constraint Verification

### AC-3.1: Google OR-Tools Constraint Solving (FR-3.1, FR-3.2)
* **AC-3.1.1:** Every advisory plan generated by the Planning Agent must be validated by the OR-Tools solver. The solver execution time must be under 500 milliseconds.
* **AC-3.1.2:** The solver must reject recommendations where fertilizer requirements exceed the soil's carrying capacity or where water consumption exceeds the local water drawdown quota by >5%.
* **AC-3.1.3:** If the solver fails to find a valid allocation (infeasible state), the system must raise an exception, log the state variables to MongoDB, and place the farmer profile in the manual queue of a regional Key Account Manager (KAM) within 2 seconds.

---

## 4. Delivery & Multilingual Voice (IVR & Mobile)

### AC-4.1: TTS and ASR Performance (FR-4.1)
* **AC-4.1.1:** Speech-to-Text conversion (ASR) using Whisper for regional Indian languages must achieve a Word Error Rate (WER) of < 15% under moderate background noise (e.g., wind, pump motors).
* **AC-4.1.2:** Text-to-Speech (TTS) voice generation via Coqui TTS must produce a natural-sounding audio stream with a Mean Opinion Score (MOS) score of >= 4.0.

### AC-4.2: Automated Weather alerts via IVR (US-1.3)
* **AC-4.2.1:** Proactive alert tasks triggered by IMD weather anomalies (e.g., frost risk, heavy rainfall warnings) must generate outbound Twilio calls within 5 minutes of event detection.
* **AC-4.2.2:** In case of a busy tone or unanswered call, the system must execute retry logic: 3 attempts at 10-minute intervals. If all retries fail, it must drop back to sending a localized SMS alert.

---

## 5. Security, Privacy & Performance

### AC-5.1: DPDP Compliance & Data Encryption (NFR-3.1, NFR-3.2, NFR-3.3)
* **AC-5.1.1:** The database must not persist or retrieve any farmer data without checking the `user_consent` table. Consent must be authenticated via Aadhaar OTP and renewed every 12 months.
* **AC-5.1.2:** All user PII data (phone number, coordinates, land titles) must be encrypted at rest using AES-256 and in transit using TLS 1.3.
* **AC-5.1.3:** All microservice endpoints must require JWT verification issued by the central Auth Service.
* **AC-5.1.4:** Data center location configurations in IaC (Terraform) must strictly restrict resources to the `ap-south-1` (Mumbai) and `ap-south-2` (Hyderabad) AWS regions.
